# Дашборд анализа растительности на территории пожаров

Веб-приложение для анализа вегетационных индексов на территории гарей 2005 года (Иркутская область).

## Стек

- **Бэкенд**: Python 3.11+, FastAPI, GeoPandas
- **Фронтенд**: Next.js 16, React 19, Recharts, Leaflet, Tailwind CSS

## Структура проекта

```
Dashboard/

│   ├── main.py
```

## API эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/fires` | GeoJSON полигонов пожаров |
| GET | `/api/fires/meta` | Метаданные пожаров (CSV-данные) |
| GET | `/api/lesnichestva` | GeoJSON лесничеств |
| GET | `/api/filters` | Доступные значения для фильтров |
| GET | `/api/vi/aggregate` | Агрегированная динамика VI по всем пожарам |
| GET | `/api/vi/by-fire/{id}` | Динамика VI для одного пожара |
| GET | `/api/vi/multi-fire` | Динамика VI для набора пожаров |
| GET | `/api/veg-areas/aggregate` | Динамика растительного покрова (сводно) |
| GET | `/api/veg-areas/by-fire/{id}` | Растительный покров для одного пожара |
| GET | `/api/raster/available?fire_id=` | Доступные растры пожара (год, сенсор, **WGS84-границы**) |
| GET | `/api/raster/{id}/{year}/{band}.png` | Растр VI в цвете (PNG); заголовок `X-Bounds` — границы для наложения |

## Функциональность

- Карта пожаров с полигонами + слой лесничеств
- Кросс-фильтрация: клик на полигон выбирает пожар, все графики пересчитываются
- **Карта вегетационного индекса прямо на карте** — при выборе одного пожара его
  снимок в выбранном VI (NDVI/NBR/…) накладывается на Leaflet как геореференнный
  слой по реальным границам растра; в панели — переключатель слоя, выбор года
  снимка (с отметкой года пожара) и регулировка прозрачности
- Галерея «Снимки гари в VI» — миниатюры по всем индексам и годам с увеличением
- Выбор вегетационного индекса (NDVI, EVI, NBR, NBR2, BAI, NDWI, SAVI)
- Выбор агрегации (median / max / min)
- Фильтр по периоду (до пожара / после / весь)
- Фильтр по типу растительности
- Фильтр по площади гари
- Динамика VI по годам относительно пожара (−5 … +20 лет)
- Диаграмма чувствительности — отклонение от допожарного среднего в %
- Стековая диаграмма площадей растительного покрова по типам

## Масштабирование и подключение новых данных

Прототип построен на данных по гарям сентября 2005 г., но структура заведомо
рассчитана на расширение (другие годы, сенсоры, индексы). Что уже сделано для
масштабируемости и что нужно разработчику, чтобы «подцепить» новые данные.

### Что уже не привязано к 2005 году

- **Год пожара вычисляется из данных** (`dt_first` гари), а не захардкожен — все
  подписи, разрезы «до/после» и линия пожара подстроятся автоматически.
- **Растры подхватываются автоматически.** Бэкенд при старте сканирует
  `Data/fire_rasters/fire_rasters/` по шаблону имени
  `fire<id>_<year>_indices_<sensor>.tif` (`RASTER_RE` в `backend/main.py`) и
  строит индекс `fire_id → [{year, sensor}]`. Достаточно положить новые `.tif`
  с таким именем — отдельный код не нужен. Границы для наложения на карту
  считаются из самого растра (любой CRS → WGS84), правок не требуют.
- **Фильтры строятся из данных.** Списки индексов, типов растительности,
  лесничеств, диапазон площадей отдаёт `/api/filters` из самих CSV/GeoJSON, а не
  из констант — новые значения появятся в UI сами.

### Шаги для добавления нового набора (напр. пожары 2010 г.)

1. **Метаданные и VI-ряды.** Сложить CSV в каталог индексов
   (`vi_long_format.csv` со столбцами `fire_id, year, years_since_fire, sensor,
   frname, veg_name, value, index, agg`; `dashboard_fires_metadata.csv` с
   `fire_id, frname, Area, dt_first, dt_last`; `areas_vegetation_in_fires_*.csv`).
   Текущие пути заданы константой `VI_DIR` в `backend/main.py` — для нескольких
   лет стоит вынести её в список каталогов и склеивать при загрузке (точка
   расширения отмечена в коде).
2. **Полигоны гарей.** Шейпфайлы — в `Data/fires_modis_2001_2024/`; собрать в
   GeoJSON (как `fires_2005_vi.geojson`) со свойствами `fire_id, Area, frname,
   dt_first, …`. Карта и кросс-фильтрация заработают без изменений фронта.
3. **Растры VI.** Положить `.tif` в `Data/fire_rasters/fire_rasters/` с именем
   `fire<id>_<year>_indices_<sensor>.tif`. Порядок каналов в файле должен
   совпадать с `BAND_NAMES` в `backend/main.py` (`NDVI, NBR, NBR2, EVI, SAVI,
   BAI, NDWI`). Галерея и наложение на карту появятся автоматически.
4. **Перезапустить бэкенд** — индекс растров и кэши пересоберутся на старте.

### Идеи для дальнейшего развития

- Вынести `VI_DIR` в список каталогов / манифест датасетов (`datasets.json`),
  чтобы годы подключались декларативно, без правки кода.
- Подготовка вегетационных индексов из «сырых» сцен для лет, которых ещё нет в
  наборе (необязательная задача из ТЗ).
- Кэширование отрендеренных PNG на диск (сейчас `lru_cache` в памяти).
